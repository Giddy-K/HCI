Part 1

Project Overview

The Kenya Revenue Authority's iTax system serves over 7 million taxpayers but suffers from significant usability issues. Our project applies Human-Computer Interaction principles to redesign the system, addressing 6 critical problems that frustrate users daily.

68% of users struggle to complete basic tasks
35-45 min average task completion time (benchmark: 10-15 min)
35% error rate during filing
10,000+ monthly helpline calls for basic navigation

The 6 Critical Issues We're Solving

01

Poor Mobile Usability

Interface doesn't adapt well to smaller screens. 58% of users access via mobile but system not optimized.

  • Tiny touch targets (24x24px vs 44x44px recommended)
  • Horizontal scrolling required
  • Forms break on small screens
02

UI/UX Overcrowded Screens

Too much information, poor visual hierarchy. 20+ options on homepage overwhelm users.

  • 40+ menu items causing navigation confusion
  • Inconsistent layouts across pages
  • Excessive text and jargon
03

Poor Accessibility & Inclusion

No font adjustment, dark mode, or screen reader support. WCAG violations throughout.

  • Color contrast ratios as low as 2.1:1 (need 4.5:1)
  • No keyboard navigation support
  • Fixed font sizes exclude visually impaired
04

Slow Performance

Long loading times, timeouts during peak periods. Average 8.4s page load (benchmark: <3s).

  • Initial load time of 8-12 seconds
  • Monolithic bundles increase load time
  • No caching strategy implemented
05

Poor Alert & Notification System

Unclear error messages like "Error 500" with no explanation. No progress indicators.

  • Cryptic technical error messages
  • No loading indicators during processing
  • No auto-save functionality
06

Lack of Guidance & Onboarding

No tutorials or clear instructions. 100% of first-timers need external help.

  • No contextual help or tooltips
  • Complex forms without explanations
  • Difficult to find support resources

Part 3

Design Alternatives

We explored three distinct design approaches, each addressing different user needs and scenarios. User testing informed our final hybrid solution.

Alternative 1

Mobile-First Progressive Approach

Design Rationale

Prioritizes mobile users (58% of traffic) with progressive disclosure to reduce cognitive overload. Multi-step wizards break complex forms into manageable chunks.

Key Features

  • Card-based interface with 4 primary actions
  • Bottom navigation for thumb-friendly access
  • One question per screen (mobile)
  • Auto-save after each step
  • Step indicator (Step 3 of 8)

User Scenario - Sarah filing return on mobile:

Opens iTax → Sees 4 main cards → Taps "File Income Tax" → Wizard starts → Question 1: "Were you employed?" → Yes → Question 2: "Rental income?" → Enters data → System auto-calculates → Reviews summary → Submits → Downloads receipt. Time: 12 minutes (vs 45 currently)

User Feedback (25 users tested)

92% found navigation easier 88% completed task first try "Much less overwhelming" 16% wanted full form view (power users)
Excellent mobile experience High completion rate Extra clicks for power users
Alternative 2

Dashboard-Centric Intelligence

Design Rationale

Personalized dashboard as central hub with predictive assistance. System proactively guides users based on deadlines and history.

Key Features

  • Smart dashboard showing "What you need to do"
  • Deadline countdown alerts
  • One-click actions with pre-filled data
  • AI chatbot for natural language queries
  • Smart suggestions based on user pattern

User Scenario - David filing VAT:

Logs in → Dashboard shows "VAT Return due in 2 days" → Clicks "Quick File VAT" → Form pre-filled from last month → Updates changed values → System validates → Submits → Auto-generates payment slip → One-click M-PESA payment. Time: 8 minutes (vs 60)

User Feedback (30 users tested)

94% loved personalized dashboard 89% found chatbot helpful 22% had privacy concerns about pre-filling
Massive time savings Proactive guidance Privacy concerns Complex backend required
Alternative 3

Modular Progressive Disclosure

Design Rationale

Adaptive interface with Beginner/Standard/Expert modes. Users can switch modes based on expertise level.

Key Features

  • Beginner mode: Simple, guided, tooltips everywhere
  • Standard mode: Balanced interface
  • Expert mode: Keyboard shortcuts, batch operations
  • Customizable dashboard
  • Progressive disclosure (show essential, hide advanced)

User Scenario - Grace (Expert mode) filing 10 clients:

Switches to Expert mode → Selects first client → Keyboard shortcut "Ctrl+V" → All fields visible → Tab navigation → Pastes from Excel → Ctrl+S to submit → Repeats for 9 more clients → Batch downloads receipts. Time: 50 min total (vs 600 min)

User Feedback (35 users tested)

96% of power users loved Expert mode 91% of beginners preferred Beginner mode "Best of both worlds"
Serves all user levels Power users very efficient Complex to develop (3 interfaces)

Final Design Selection: Hybrid Approach

After extensive testing with 90 users across all three alternatives, we chose a HYBRID APPROACH combining the best elements:

From Alt 1 Mobile-first responsive design
From Alt 2 Dashboard with quick actions & smart suggestions
From Alt 3 Task-based navigation & progressive disclosure

Result: Our Figma design balances mobile usability, clear information hierarchy, and intuitive navigation for all user types.
